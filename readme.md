Оценка текущего состояния проекта
Проект структурирован хорошо, соблюдены принципы модульности, централизованной конфигурации и предзагрузки ресурсов. Код выглядит чистым и поддерживаемым. Однако есть несколько моментов, которые стоит улучшить до добавления смены фона по уровням:

1. Жёсткая привязка фона к одному изображению
Сейчас фон «Горы» (класс mountains.js) инициализируется один раз при старте и не умеет переключать текстуру по уровням. Если просто заменить mountains-bg.png на factories-bg.png – горы исчезнут на всех уровнях.
Нужно: сделать фон переключаемым (через данные уровня или через метод setLevel).

2. Дублирование логики фоновых слоёв
world.js содержит прямые ссылки на sky и mountains, но rocks и skybackground живут отдельно. Это затрудняет добавление новых фоновых типов.
Лучше: унифицировать все фоновые элементы через систему слоёв (layers.js), но для текущей задачи это опционально.

3. Отсутствие типа фона в данных уровня
В level1.js / level2.js нет поля backgroundType или backgroundImage. Придётся хардкодить проверку level.number === 2, что не масштабируется.

4. Предзагрузка ресурсов не включает второй фон
Сейчас загружается только mountains-bg.png. Для заводов нужно добавить factories-bg.png в loadAllImages().

5. Потенциальная проблема с тайлингом/параллаксом
Если класс mountains.js реализует тайлинг на основе размера изображения, то у заводов может быть другая ширина/высота – надо проверить совместимость.

Нужен ли рефакторинг?
Лёгкий рефакторинг – да, но без переписывания архитектуры. Достаточно:

Добавить в объект уровня поле backgroundImage (или backgroundType).

В world.setLevel() обновить изображение у mountains (или пересоздать объект).

Расширить loadAllImages() для загрузки второго фона.

Убедиться, что mountains.js умеет динамически менять спрайт.

Если вы не хотите трогать класс mountains, можно создать отдельный класс Factories и подменять его в world по условию – но это менее элегантно.

Какие файлы нужны для анализа (если решите делать рефакторинг)
js/background/mountains.js – чтобы понять, как он использует изображение и можно ли добавить метод setImage(src).

js/world.js – посмотреть, как создаётся и используется mountains.

js/main.js – секция загрузки изображений.

levels/level1.js и levels/level2.js – чтобы добавить поле background.

js/config.js – возможно, вынести пути к фоновым изображениям.

Предлагаемый план действий (без глубокого рефакторинга)
Добавить в config.js пути:
BACKGROUND_MOUNTAINS: 'assets/mountains/mountains-bg.png'
BACKGROUND_FACTORIES: 'assets/factories/factories-bg.png'

В loadAllImages() загрузить оба изображения.

В классе Mountains добавить метод setImage(imageSrc) или setImageObject(imgElement).

В world.setLevel() вызывать this.mountains.setImage(level.backgroundImage).

В level2.js добавить поле backgroundImage: CONFIG.BACKGROUND_FACTORIES (или строку).

Убедиться, что координаты заводов на уровне визуально совпадают с горами (Y-позиция, масштаб). При необходимости скорректировать в mountains.js параметры отрисовки.
